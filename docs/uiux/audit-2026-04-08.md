# UI/UX Audit Cafe-X (2026-04-08)

## Scope
- Customer (Next.js)
- Admin (Next.js)
- POS (Flutter)

## Ringkasan Pain Points
### Customer
- Hierarki visual belum konsisten: hero, filter, promo, dan status masih “bertabrakan” fokus.
- Cart drawer sudah ada tapi belum terasa “priority action” (CTA kurang dominan).
- Sticky categories belum konsisten saat scroll panjang (baru di hero/panel).
- Skeleton loading masih generik; belum match tone brand.
- Error states token/expired belum punya visual state khusus.

### Admin
- Sidebar dan topbar sudah modern, tapi tone visual berbeda dengan customer (warna & kontras).
- Kontras visual action kritikal (billing, reset demo) belum dibedakan.
- Dashboard KPI masih terasa “flat”; tidak ada trend indicator.
- Status badge dan table density belum konsisten antar halaman.

### POS (Flutter)
- Transisi antar layar masih hard cut (tanpa animasi ringan).
- Offline/sync banner ada, tapi prioritas visual kurang jelas saat error.
- Conflict resolution masih teks; belum terasa sebagai flow guided.
- Payment flow sudah cepat, tapi CTA/shortcut highlight belum konsisten.

## Bottlenecks (Flow)
- Customer: input meja → menu → cart → order-status masih butuh lebih sedikit langkah visual dan guidance.
- Admin: onboarding outlet pertama belum ada (user harus paham struktur sendiri).
- POS: konflik sync membuat kasir bingung karena tidak ada stepper/decision flow.

## Inkonsistensi Tokens/Components
- Typography: customer pakai headline khas, admin masih standard (perlu unify).
- Buttons: bentuk, gradient, dan hover berbeda antar app.
- Cards: radius dan shadow beda antar app.
- Badges: status warna tidak konsisten antar admin/pos.

## Prioritas Remodel (Top 8)
1. Customer Menu (sticky categories + quick add + hero hierarchy)
2. Customer Cart (drawer/bottom sheet + CTA kuat)
3. Customer Order Status (stepper + ETA + error visual states)
4. Admin Dashboard (KPI cards + trend + contrast)
5. Admin Subscription/Billing v1 (manual invoice)
6. Admin Demo Mode reset (sales enablement)
7. POS Order → Cart → Payment (smooth transitions + CTA highlight)
8. POS Offline/Sync Banner + Conflict Resolution flow

## Quick Wins
- Unified color tokens across apps.
- Micro interaction: hover/press/scale for CTA.
- Skeleton loaders matching brand gradients.
